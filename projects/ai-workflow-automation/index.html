Overview

What I built

Personal automation work connecting LLM APIs, webhooks, and external services with n8n. The workflows automate data processing, smart notifications, AI-generated content flows, conditional routing, and error-aware execution.

Converted manual multi-step work into repeatable workflows with visible control points and failure handling.

Focused on practical automation: fast iteration, clear inputs/outputs, and workflows that can be inspected instead of running as hidden magic.

Highlights

What makes it notable

Webhook-driven execution

Workflows start from explicit triggers and move through LLM calls, conditions, retries, and notifications.

Visible control points

The automation is modeled so inputs, branches, outputs, and failure paths are inspectable.

AI-assisted delivery

LLM APIs are used as part of a workflow system rather than as untracked manual prompts.

Engineering

Decisions and constraints

Decision 01

Avoiding hidden scripts

n8n makes the route from trigger to output visible, which helps debug and maintain automation behavior.

Decision 02

Keeping failure paths explicit

Retry and notification branches keep failed automation runs observable instead of silently dropping work.

Decision 03

Separating routing from generation

Conditions and service steps decide where data goes; LLM calls handle the specific generation or reasoning task.
